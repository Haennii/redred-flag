# RedFlag — 은행 재무제표 이상징후 분석 대시보드

4대 시중은행(KB국민·신한·하나·우리)의 재무제표를 분석하여 감사위험을 조기에 탐지하는 대시보드입니다.

## 개요

회계법인 감사부문에서 실제로 사용하는 관점으로 은행 재무지표의 이상징후를 자동 탐지하고 시각화합니다.

## 탐지 지표

| 지표 | 기준 | 감사 관점 |
|------|------|-----------|
| 자기자본이익률 (ROE) | < 8.5% 주의 | ROE 하락 시 이익조정 유인 증가, 자본 효율성 저하 |
| 순이자마진 (NIM) | < 1.6% 주의 | 금리리스크 노출, 자산·부채 만기 불일치(ALM) 점검 필요 |
| 보통주자본비율 (CET1) | < 8.5% 위험 | Basel III 손실흡수 자본 적정성, 규제 미달 시 배당 제한 |
| 순영업수익 | 전년비 -10% 주의 | 이자이익+비이자이익 합산 핵심 수익력 저하 신호 |
| 비이자이익 | 전년비 -15% 주의 | 수익 다변화 약화, 금리 변동 취약성 증가 |
| 대손비용 | 전년비 +30% 주의 | 부실 대출 증가 또는 ECL 모델 가정 변경 신호 |
| 대손비용률 | > 0.3% 주의 | 대출 포트폴리오 자산건전성 선행지표 |

## 기술 스택

- **Frontend**: React + TypeScript + Vite
- **차트**: Recharts
- **스타일**: Tailwind CSS
- **데이터**: DART 전자공시 OpenAPI (2023-2024 실데이터) + 사업보고서 기반 추정치 (2020-2022)

## 실행 방법

```bash
# 의존성 설치
npm install

# DART API 키 설정 (opendart.fss.or.kr 무료 발급)
cp .env.example .env
# .env 파일에 VITE_DART_API_KEY 입력

# 개발 서버 실행
npm run dev
```

> API 키가 없으면 사업보고서 공시 기반 데이터로 자동 동작합니다.

## 데이터 출처

- DART 전자공시시스템 (opendart.fss.or.kr) — 별도재무제표 (fnlttSinglAcntAll)
- 각 은행 사업보고서 — 보통주자본비율(CET1) 등 감독당국 보고 지표
- 금융감독원 금융통계정보시스템 — 업계 평균 벤치마크

## 분석 대상

KB국민은행 · 신한은행 · 하나은행 · 우리은행 (2020–2024)
