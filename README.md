# RedFlag — 은행 재무제표 이상징후 분석 대시보드

4대 시중은행(KB국민·신한·하나·우리)의 재무제표를 분석하여 감사위험을 조기에 탐지하는 대시보드입니다.

## 개요

회계법인 감사부문에서 실제로 사용하는 관점으로 은행 재무지표의 이상징후를 자동 탐지하고 시각화합니다.

## 탐지 지표 (Red Flags)

| 지표 | 기준 | 감사 관점 |
|------|------|-----------|
| BIS 자기자본비율 | < 10.5% (Basel III) | 자본적정성 위협, 조기시정조치(PCA) 가능성 |
| 고정이하여신비율 (NPL) | > 0.8% | 대손충당금 추가 적립 필요성, 이익 과대계상 의심 |
| 대손충당금 적립률 | < 100% | 이익조정(Earnings Management) 위험 |
| ROA | < 0.5% | 자산 수익창출 능력 저하 |
| 순이자마진 (NIM) | < 1.6% | 금리리스크 노출, ALM 점검 필요 |
| 예대율 | > 100% | 단기 유동성 위기 취약성 |
| 대출 급성장 | > 15% YoY | 심사기준 완화 의심, 향후 NPL 상승 선행지표 |
| 당기순이익 급감 | < -20% YoY | 대손충당금 대규모 전입 또는 이전연도 이익 과대계상 |

## 기술 스택

- **Frontend**: React + TypeScript + Vite
- **차트**: Recharts
- **스타일**: Tailwind CSS
- **데이터**: DART 전자공시 OpenAPI + 사업보고서 기반 추정치

## 실행 방법

```bash
# 의존성 설치
npm install

# DART API 키 설정 (opendart.fss.or.kr 무료 발급)
cp .env.example .env
# .env 파일에 VITE_DART_API_KEY 입력

# 개발 서버 실행
npm run dev
```

> API 키가 없으면 사업보고서 공시 기반 데이터로 자동 동작합니다.

## 데이터 출처

- DART 전자공시시스템 (opendart.fss.or.kr) — 재무제표 원시 데이터
- 각 금융지주 사업보고서 — BIS비율, NPL비율 등 감독당국 보고 지표
- 금융감독원 금융통계정보시스템 — 업계 평균 벤치마크
